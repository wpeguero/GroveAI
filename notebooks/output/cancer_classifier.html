Classifier for Cancer¶

This is the basics for using the pyblight library to train a machine learnign model for cancer classification. The purpose of this library is to develop efficient tumor classifiers that will liberate radiologists from having to diagnose large number of patients based on image data all the while focusing on the rarer cases.

Purpose of the notebook¶

  • Show how machine learning algorithms and the pytorch library can be used to classify tumors
  • How to set up data from the cancer imaging archive (tcia) for training ML models
  • The result of images
  • Process images for normalization

Business Understanding¶

  • Mortality rate for cancer has been increasing gradually throughout time
  • Three general areas are used to measure the progress of the fight against cancer
    1. direct measures (mortality, incidence, and survival)
    2. portents of change (reductions in tobacco use)
    3. advances in knowledge that may have an effect on the future

Data Understanding¶

  • Cancer imaging data is the main source used by radiologists to diagnose tumors.
  • The image itself is developed through a CT scan
    • This allows one to highlight certain features within the body based on the water content (which shows up as brighter pixels within the image).
  • Main data form is DICOM files
    • These files may contain a set of images that symbolize a different slice from the DICOM file.

Data Preparation¶

  • To use the image data, one first has to load the images from the dicom file.
  • The data may be obfuscated within a set of folders that make organizing the images more difficult.
  • We extract the data using built-in functions from the pyblight library
  • A csv file was created containing the categorical data from the existing dataset csv files and the paths to all of the dicom files within the dataset.
    • This is used to load the images for training.

The DICOM Files¶

The file type containing the images of the dataset are called Digital Imaging and Communications in Medicine (DICOM for short). This is meant to be a standard for medical images and the metadata associated with the aforementioned medical image. Each sample in the dataset may contain a myriad of images that represent a slice of the patient.

We first begin by loading the dicom file whose path was extracted from the dataframe using the pydicom library:

As shown above, the loaded diucom file contains textual data associated with the medical image itself. In a hospital setting, the dicom file will contain personal patient information (such as age, height, weight, etc). Due to the anonymization process for the patient's safety, many fields have been removed. Now that hte dicom file has been loaded, one can work towards extracting the image(s).

As shown above, the image is currently of a very high resolution (2294, 1914). This is not ideal for machine learning as the amount of data extracted from each individual pixel scales the model to a very large size (possibly within the terabites level). Ideally, one would extract region(s) of of interest from the image which would be smaller than the image resolution and make the model lighter. For now, the image will be rescaled to a more proportionate size (512, 512).
